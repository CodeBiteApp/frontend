#### **(1-A)**

문제: **Java 데이터 타입**에 대한 설명으로 옳은 것은?

1. Java의 플랫폼 독립성은 JVM이 제공하고 실행 성능은 JIT 컴파일러가 보완한다.
   (오답 - `Java 프로그래밍이란`의 `feature`, row=3)
2. Java 기본형에는 byte, short, int, long, float, double, char, boolean 8가지가 있다.
   (정답 - `기본형`의 `specification`, row=19)
3. Java EE는 웹과 엔터프라이즈 기능(API, 트랜잭션, DI 등)을 포함한 확장 플랫폼이다.
   (오답 - `Java EE`의 `definition`, row=6)
4. 캡슐화 관점에서 접근 제어자는 가능한 한 좁은 범위로 열어두는 것이 권장된다.
   (오답 - `접근 제어자`의 `best_practice`, row=12)

정답: 2

#### (1-B)

문제: **접근 제어자**에 대한 설명으로 옳은 것은?

1. Java는 컴파일 후 바이트코드를 실행(JVM 필요)하며 자동 메모리 관리(GC)를 사용한다.
   (오답 - `Java와 C/C++ 차이`의 `comparison`, row=7 일부)
2. protected는 같은 패키지와 상속관계에서 접근 가능한 접근 제어자이다.
   (정답 - `protected`의 `definition`, row=14)
3. 기본형 변수는 객체의 참조값(주소 개념)을 보관한다.
   (오답 - `Java 데이터 타입`의 `comparison` 반대 진술, row=17 반전)
4. Java SE는 웹과 엔터프라이즈 기능을 포함한 확장 플랫폼이다.
   (오답 - `Java SE/EE` 관계 뒤바뀜, row=5/6 반전)
